## Overview

Civic never asks for API keys or secrets directly in chat. All credential collection happens through a **secure authorization flow** when a tool connection is first established, preventing secrets from being exposed in conversation logs.

## Client Secrets

When connecting programmatically (e.g., via the Agent API), client secrets are managed through your dashboard.

<Info>
To generate a client secret, log into your dashboard at [auth.civic.com](https://auth.civic.com), navigate to the **Security** tab, and click **Generate Client Secret**.

**Important:** The client secret is only displayed once upon generation. Copy and securely store it immediately. You can always regenerate a new client secret if needed.
</Info>

<Warning>
**Security Requirements:** Client secrets must be stored securely and never exposed in client-side code. Use environment variables or a secure secret management system. Always use HTTPS in production to protect credentials in transit.
</Warning>

## Authentication Flows

Civic supports three authentication modes depending on your security requirements:

<CardGroup cols={3}>
  <Card title="PKCE Only" icon="lock">
    **Best for:** Standard web and mobile apps.
    No client secret needed — PKCE is handled entirely by the Civic Auth SDK.
  </Card>
  <Card title="Client Secret Only" icon="key">
    **Best for:** Legacy server-side applications that do not support PKCE.
    Uses traditional OAuth 2.0 authorization code flow.
  </Card>
  <Card title="PKCE + Client Secret" icon="shield">
    **Best for:** Maximum security for confidential clients.
    - Financial or healthcare systems
    - Enterprise backend services
    - Combines PKCE with client secret for dual verification
  </Card>
</CardGroup>

## Agent Token

For agent connections to `nexus.civic.com`, pass a `CIVIC_TOKEN` in the `Authorization` header. This keeps secrets server-side and out of the agent's context.

```http
POST https://nexus.civic.com/hub/mcp
Content-Type: application/json
Authorization: Bearer <civic-token>
```

If your organization has multiple profiles configured, include the `x-civic-profile` header:

```http
x-civic-profile: default
```

## Summary

| Method | Storage Requirement | Use Case |
|--------|-------------------|----------|
| PKCE Only | None | Web/mobile apps |
| Client Secret | Environment variable or secret manager | Legacy server-side apps |
| PKCE + Client Secret | Environment variable or secret manager | High-security backends |
| Agent Token (`CIVIC_TOKEN`) | Environment variable | Agent/API integrations |
