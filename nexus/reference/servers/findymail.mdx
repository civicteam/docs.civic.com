## Overview

The Findymail server enables AI assistants to interact with [Findymail](https://findymail.com), providing email finding, verification, and contact discovery capabilities for sales teams and business development professionals.

## How to Add Findymail

<Steps>
  <Step title="Create Findymail Account">
    1. Go to [https://findymail.com](https://findymail.com)
    2. Sign up for an account or log in to your existing account
  </Step>
  <Step title="Generate API Key">
    1. Navigate to your account settings
    2. Go to **API Settings**
    3. Generate a new API key
    4. Copy the API key for use in Nexus
  </Step>
  <Step title="Connect to Nexus">
    1. Add the Findymail server to your Nexus environment
    2. Enter your API key when prompted
  </Step>
  <Step title="Test Connection">
    Start with a simple command like "Verify if john.doe@company.com is a valid email address" to verify the connection works properly.
  </Step>
</Steps>

<Tip>
Findymail uses separate credit pools for "finder" operations and "verifier" operations. Monitor your credit usage to avoid unexpected limits.
</Tip>

## Use Cases

### Email Verification
"Verify if john.doe@company.com is a valid email address" - Quickly validate email addresses before sending outreach.

### Contact Discovery
"Find the email for Sarah Johnson at salesforce.com" - Locate specific contacts at target companies.

### Executive Outreach
"What's the email address for the CTO at stripe.com?" - Find decision-makers by role.

### Lead Validation
"Check if this email is deliverable: contact@example.com" - Ensure email addresses are valid before adding to CRM.

### LinkedIn Research
"Get details from this LinkedIn profile: linkedin.com/in/marcbenioff" - Extract contact data from professional profiles.

<Note>
Findymail separates credits into "finder" credits and "verifier" credits. Each operation type consumes from its respective pool.
</Note>

<Warning>
**Known Limitations:**
- **Phone finder** returns same data as email finder, not actual phone numbers
- **LinkedIn URL finder** returns company pages instead of personal profiles
- **Company data** has sparse coverage, even for major companies
- **No confidence scores** to assess result reliability
- **Variable success rates** - legitimate queries may return null results
- **No credit monitoring tool** available via MCP
- **Limited error differentiation** - cannot distinguish between "not found" and rate limits
</Warning>
