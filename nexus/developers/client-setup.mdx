## Overview

## Quick Start (2 Options)

### Option 1: Direct HTTP Connection

<Steps>
  <Step title="Get MCP URL">
    Visit [nexus.civic.com](https://nexus.civic.com) and copy your personal MCP endpoint URL
  </Step>
  <Step title="Configure Client">
    Add the HTTP endpoint directly to your development tool's MCP configuration
  </Step>
  <Step title="Authenticate">
    Follow OAuth prompts when first using any server - tokens are securely stored
  </Step>
</Steps>

### Option 2: Hub Bridge (Local Proxy)

<Steps>
  <Step title="Configuration">
    Add `npx -y @civic/hub-bridge` as your MCP server command in client settings
  </Step>
  <Step title="Authentication">
    Run any MCP command - OAuth browser window opens automatically for first-time setup
  </Step>
  <Step title="Usage">
    Hub Bridge handles authentication and routes requests to your selected Civic Nexus servers
  </Step>
</Steps>

<Info>
**Performance:** HTTP endpoints are faster. **Compatibility:** Hub Bridge works with more clients that don't support HTTP endpoints yet.
</Info>

## Cursor Setup {#cursor}

**Recommended:** Use the one-click install for fastest setup.

<Steps>
  <Step title="Create Config File">
    Create .cursor/mcp.json in your project root:
    ```json
    {
      "mcpServers": {
        "civic-nexus": {
          "command": "npx",
          "args": ["-y", "@civic/hub-bridge"]
        }
      }
    }
    ```
  </Step>
  <Step title="Restart Cursor">
    Restart Cursor to load the MCP configuration
  </Step>
</Steps>

## VS Code Setup {#vscode}

**VS Code now supports both remote HTTP endpoints and Hub Bridge.**

<Steps>
  <Step title="Get MCP URL">
    Visit [nexus.civic.com](https://nexus.civic.com) and copy your MCP endpoint URL
  </Step>
  <Step title="Add to VS Code">
    Open VS Code settings.json and add:
    ```json
    "mcp.servers": {
      "civic-nexus": {
        "url": "YOUR_MCP_ENDPOINT_URL_HERE"
      }
    }
    ```
  </Step>
  <Step title="Test Connection">
    Open any file and try using MCP tools through VS Code's interface
  </Step>
</Steps>

<AccordionGroup>
  <Accordion title="Node.js not found">
    **Error:** `npx: command not found` or `node: command not found`
    
    **Solution:** Install Node.js 18+ from [nodejs.org](https://nodejs.org)
    
    Verify installation:
    ```bash
    node --version  # Should show v18+ 
    npm --version   # Should show 8+
    ```
  </Accordion>
  <Accordion title="Hub Bridge fails to start">
    **Error:** Hub Bridge exits immediately or shows connection errors
    
    **Solutions:**
    1. **Check Node.js version:** Must be 18 or higher
    2. **Clear npm cache:** Run `npm cache clean --force`
    3. **Manual install:** Run `npm install -g @civic/hub-bridge` then use `hub-bridge` instead of `npx -y @civic/hub-bridge`
    4. **Firewall/Network:** Ensure your network allows connections to `nexus.civic.com`
    5. **Proxy users:** Configure npm proxy settings if behind corporate firewall
    
    **Debug Mode:**
    ```bash
    DEBUG=* npx -y @civic/hub-bridge
    ```
    
    This will show detailed logs to help identify the issue.
  </Accordion>
</AccordionGroup>

## JetBrains IDEs Setup {#jetbrains}

**Supported:** IntelliJ IDEA, PyCharm, WebStorm, PhpStorm, GoLand, etc.

<Steps>
  <Step title="Install MCP Plugin">
    1. Go to **File > Settings > Plugins**
    2. Search for "MCP" in the marketplace
    3. Install the **Model Context Protocol** plugin
    4. Restart your IDE
  </Step>
  <Step title="Configure MCP Server">
    1. Go to **File > Settings > Tools > MCP**
    2. Add new server configuration:
       - **Name:** Civic Nexus
       - **Command:** `npx`
       - **Arguments:** `-y @civic/hub-bridge`
    3. Save configuration
  </Step>
  <Step title="Enable and Test">
    1. Enable the Civic Nexus server in MCP settings
    2. Test connection by opening any file and accessing MCP tools
  </Step>
</Steps>

## Best Practices

### Development Workflow

1. **Start simple** - Use one server first (like GitHub or Slack)
2. **Test incrementally** - Add one server at a time
3. **Monitor usage** - Check your Civic Nexus dashboard for activity
4. **Update regularly** - Keep Hub Bridge updated with `npm update -g @civic/hub-bridge`

### Team Collaboration

**For teams using the same tools:**

1. **Commit config files** to version control (they don't contain secrets)
2. **Document setup** in your project README
3. **Use consistent naming** across team members
4. **Share tool selections** - use same Civic Nexus server selection

### Security Considerations

- **Hub Bridge runs locally** - tokens never leave your machine except to make API calls
- **Tokens are encrypted** using OS-level key storage
- **OAuth scopes are minimal** - only request necessary permissions
- **Revoke access anytime** at nexus.civic.com
